# Спецификация требований Chatty — версия 2.0

## О проекте

Этот репозиторий содержит полную спецификацию требований (Software Requirements Specification) ко второй версии веб-приложения **Chatty** — платформы для изучения английского языка.

Я самостоятельно разработал данный документ в роли системного аналитика. Цель работы — создать чёткое, полное и структурированное техническое задание, которое позволит разработчикам и тестировщикам точно реализовать и проверить новую функциональность версии 2.0.

## О приложении Chatty

**Chatty** — веб-приложение, предназначенное для совместного изучения английского языка учениками и преподавателями.

Основные пользователи:

- **Ученики** — люди, изучающие английский (в среднем 2 раза в неделю)
- **Преподаватели** — преподаватели английского языка (в среднем 7 часов в день)

В первой версии приложение уже позволяло проводить уроки по расписанию, назначать домашние задания и отслеживать успеваемость.  
В версии **2.0** добавляется мощный функционал работы со словами:

- встроенный словарь с переводом, произношением, примерами и видео
- персональный словарь пользователя
- карточки для запоминания слов по системе повторения
- просмотр контекстных видео с употреблением слова

### Основные разделы спецификации

1. Введение  
   Цель документа, границы проекта, ссылки

2. Общее описание продукта  
   Контекстная диаграмма, классы пользователей, операционная среда, ограничения и предположения

3. Функции системы

   - Поиск и работа со словарем
   - Персональный словарь пользователя
   - Создание и изучение карточек
   - Просмотр видео в контексте  
     11 детализированных use case с основным сценарием, расширениями и исключениями

4. Требования к данным  
   Логическая модель данных, словарь данных, правила хранения

5. Требования к внешним интерфейсам  
   Интеграция с подключаемыми словарями и видео-библиотеками, уведомления по email

6. Атрибуты качества  
   Удобство использования (одно действие = добавление/удаление)  
   Производительность (загрузка страниц < 3 с, операции < 5 с)  
   Безопасность (приватность словарей, обязательная авторизация)  
   Доступность (99% в рабочее время)

7. Приложения  
   Глоссарий терминов  
   Бизнес-правила (в т.ч. запрет дублирования слов и соблюдение авторских прав на видео)
